# Known Issues

Актуальный список недоделок и известных проблем. См. также [Changelog.md](Changelog.md) для истории фиксов.

## Недоделки

- **R — рестарт по тапу, а не по удержанию 3 сек** — в оригинале Isaac рестарт требует удержания клавиши ~3 секунды. Сейчас у нас срабатывает по одиночному нажатию. Отложено как некритичное.
- **Нет звука** — `pygame.mixer` не используется вообще.
- **Нет анимации спрайтов** — все объекты — залитые прямоугольники, меняющие цвет (не считается анимацией в привычном смысле).
- **Полноэкранный режим всё ещё выглядит немного мыльно** — 2026-07-20 нашёл и починил реальный баг (см. Changelog: `pygame.display.Info()` вызывался ПОСЛЕ создания окна и возвращал размер окна вместо настоящего разрешения монитора, из-за чего полный экран рендерился в маленьком разрешении и растягивался ОС). После фикса fullscreen-поверхность корректно создаётся в реальном разрешении экрана (подтверждено программно: 2560×1440), но пользователь по ощущению всё ещё видит лёгкую мыльность. Возможные причины, не проверенные: масштабирование Windows (DPI scaling увеличивает интерфейс/приложения), или `pygame.transform.scale` использует не совсем целочисленный коэффициент (2.4x для 800×600→2560×1440) — целочисленный скейл (×2, ×3) обычно выглядит чётче дробного. Отложено по решению пользователя ("потом пофиксим").

## Backlog идей (не в работе)

## Мелкая неэффективность (не критично)

- `Tear.update()` создаёt новый `pygame.Surface` каждый кадр для альфа-затухания вместо переиспользования поверхности или `set_alpha()`.

## Design-декларации (не баг, но стоит знать)

- Диагональные слёзы от импульса движения реализованы с весом `momentum_weight = 0.35` — подобрано на глаз, не сверено с точным поведением оригинала (там нет открытого исходника для сверки).
