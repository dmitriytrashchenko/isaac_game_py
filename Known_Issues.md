# Known Issues

Актуальный список недоделок и известных проблем. См. также [Changelog.md](Changelog.md) для истории фиксов.

## Недоделки

- **R — рестарт по тапу, а не по удержанию 3 сек** — в оригинале Isaac рестарт требует удержания клавиши ~3 секунды. Сейчас у нас срабатывает по одиночному нажатию. Отложено как некритичное.
- **Нет звука** — `pygame.mixer` не используется вообще.
- **Нет покадровой анимации спрайтов** — есть процедурная анимация (парение предметов, мерцание факелов, частицы, screen shake — см. [[Session_2026-07-20_Visual_Overhaul]]), но сами силуэты статичны — нет цикла ходьбы/атаки.
- **Полноэкранный режим всё ещё выглядит немного мыльно** — 2026-07-20 нашёл и починил реальный баг (`pygame.display.Info()` вызывался ПОСЛЕ создания окна, возвращал неверное разрешение). После фикса fullscreen-поверхность корректно создаётся в реальном разрешении экрана (подтверждено программно: 2560×1440), но пользователь по ощущению всё ещё видит лёгкую мыльность. Возможные причины, не проверенные: DPI-масштабирование Windows, или `pygame.transform.scale` с нецелым коэффициентом (2.4x) — целочисленный скейл (×2, ×3) обычно выглядит чётче. Отложено по решению пользователя ("потом пофиксим").
- **Комнаты только прямоугольные** — рандомизация 2026-07-20 (факелы/тон/колонны, см. [[Session_2026-07-20_Visual_Overhaul]]) даёт визуальное и частично планировочное разнообразие, но произвольные формы (не прямоугольные) не реализованы — потребовало бы переписать стены/двери/коллизии/миникарту, завязанные на фиксированный прямоугольник. Отдельная большая задача, не начата.
- **Снаряды пролетают сквозь колонны** — `Room.pillars` блокируют только игрока (`Game._resolve_static_obstacle_collisions`), не тени/слёзы игрока или врагов. Не проверялось, насколько это заметно в реальной игре.
- **Валюта (глаза) без применения** — копится на `Player.eyes`, магазина или другого способа потратить пока нет.

## Backlog идей (не в работе)

- **Сеттинг: "Подземелья Забытого Короля"** — см. [Setting.md](Setting.md) для полного лора/дизайна (враги, боссы, предметы, палитра, чеклист реализации). Пока не реализовано — только зафиксирован выбор направления.
- **Магазин** — потратить накопленные глаза. Естественное продолжение введённой 2026-07-20 валюты.
- **Нерандомные формы комнат** — см. пункт в недоделках выше, требует архитектурных изменений системы комнат.

## Мелкая неэффективность (не критично)

- `Tear.update()` создаёt новый `pygame.Surface` каждый кадр для альфа-затухания вместо переиспользования поверхности или `set_alpha()`.

## Design-декларации (не баг, но стоит знать)

- Диагональные слёзы от импульса движения реализованы с весом `momentum_weight = 0.35` — подобрано на глаз, не сверено с точным поведением оригинала (там нет открытого исходника для сверки).
