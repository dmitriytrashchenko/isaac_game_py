## 2026-07-19

**Переход на следующий этаж** (`c3555fd`)
- По запросу пользователя: после победы над боссом в центре его комнаты появляется люк (фиолетовый квадрат с белой рамкой), наступив на который игрок переходит на новый этаж
- `Game._advance_floor()`: генерирует новый `Dungeon` с нуля, сбрасывает `transition`/`door_detector`, ставит игрока в центр стартовой комнаты — при этом характеристики игрока (здоровье, урон, скорость, скорострельность) НЕ сбрасываются, переносятся как есть
- `Game.floor` — счётчик этажа, отображается в UI (`Floor N` сверху по центру)
- Люк проверяется на столкновение только когда `current_room.room_type == BOSS and current_room.cleared` — не сработает, пока босс жив
- Проверено тестом: убил босса реальной коллизией слезы → люк не срабатывает, пока игрок не встанет на него специально → при заходе на люк этаж +1, статы (урон 3.5, скорость 250, здоровье 4) сохранились, новая стартовая комната — обычного типа

**Запертые двери в комнатах с врагами** (`c5fdc6f`)
- По запросу пользователя — классическая механика Isaac: пока в комнате есть живые враги, двери физически заперты и не пропускают игрока, визуально показаны закрытыми (сплошная стена + красная рамка вместо серого прохода)
- `Room.is_locked()`: `True`, если в комнате изначально были враги и она ещё не зачищена (`len(self.enemies) > 0 and not self.cleared`)
- `Room.draw()`: цикл по дверям вместо 4 дублирующихся блоков, рендер зависит от `is_locked()`
- `Game._check_room_boundaries()`: проход через дверь дополнительно гейтится `not self.current_room.is_locked()`
- Проверено тестом: попытка пройти дверь с живыми врагами — комната не меняется; после зачистки (реальная гибель через коллизию слезы) — дверь открывается и переход проходит

**Мини-карта подключена** (`7cdec1a`)

Переписан: показывает все посещённые комнаты подземелья, позиционированные относительно текущей, раскрашенные по типу (обычная/сокровищница/босс), текущая комната подсвечена белой рамкой
- `Game.draw()`: `self.ui.draw_minimap(self.screen, self.dungeon.get_minimap_data())`
- Проверено headless-рендером до и после перехода между комнатами — без ошибок, набор посещённых комнат растёт корректно

**Враги-стрелки теперь стреляют** (`3815871`)

- Новая константа `ENEMY_TEAR_SPEED = 250`
- Проверено: синий враг рядом с игроком реально стреляет и попадает (health игрока падает) за 2 секунды теста

**Баг пинг-понга переходов** (найден живым тестом пользователя)
- После перехода игрок спавнился в 4px от стены — прямо в зоне обнаружения той же двери (она уходит на ~30px вглубь комнаты из-за `zone_padding` в `DoorDetector`), из-за чего переход мгновенно запускался обратно. Отступ увеличен до 80px (`ROOM_ENTRY_MARGIN`) (`f558d7e`)

**Isaac-style управление**
- Прицел по стрелкам теперь целится по **последней нажатой** стрелке, а не по фиксированному приоритету вверх>вниз>влево>вправо — подтверждено пользователем как поведение оригинала (`c29b871`)
- Диагональные слёзы: импульс движения игрока подмешивается к направлению выстрела (вес 0.35), как в оригинале — стационарный выстрел остаётся строго по оси (`1549b7b`)
- **Настоящий баг**: стрелочные клавиши были задублированы и на движение, и на стрельбу. Из-за раздельных `if` (не `elif`-цепочки) зажатая стрелка перебивала противоположное движение WASD — "бежать влево, стрелять вправо" не работало. Убрано дублирование: движение только WASD, стрельба только стрелки/мышь (`be0bdcb`)
- Добавлено прицеливание мышью (удержание ЛКМ, 360°, приоритет над стрелками) как альтернатива, по описанию `MouseControl=1` из оригинала (`c34c4cb`)
